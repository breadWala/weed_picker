# Wi-Fi Controlled ESP32-CAM Surveillance and Robotic Platform

## Progress Documentation (Current Version)

### 1. Objective

The objective of this project is to develop a Wi-Fi controlled robotic vehicle capable of:

* Streaming live video from an ESP32-CAM to a web browser.
* Controlling the motion of a four-wheeled robot.
* Operating a robotic gripper using servo motors.
* Providing an intuitive browser-based user interface for teleoperation.

# 4. Functional Modules Completed

## 4.1 Camera Module

Implemented using the ESP32 Camera library.

Features:

* Camera initialization
* Camera configuration
* JPEG image capture
* MJPEG live streaming
* Browser-based viewing

Result:

The camera successfully streams live video over Wi-Fi.

---

## 4.2 Wi-Fi Module

Implemented using:

* WiFi.h

Responsibilities:

* Connect to local Wi-Fi
* Obtain IP address
* Start HTTP server

Result:

ESP32-CAM becomes accessible from any device on the same network.

---

## 4.3 HTTP Web Server

Implemented using the ESP32 Camera Web Server example as the base.

Responsibilities:

* Serve HTML page
* Receive browser requests
* Execute hardware control functions

Communication flow:

```
Browser

↓

HTTP GET Request

↓

ESP32 Handler

↓

Control Function
```

# 11. Current Limitations

Current implementation uses HTTP requests for every control action.

Limitations include:

* Higher latency
* New HTTP connection for each command
* Less suitable for continuous control
* No automatic stop on communication loss

# 13. Current Project Status

The project has progressed from individual hardware testing to a functional browser-controlled robotic platform. The system can stream live video while simultaneously accepting user commands for vehicle motion and gripper operation. The next development stage focuses on improving communication efficiency and responsiveness using WebSockets and adding safety mechanisms such as command timeouts.
